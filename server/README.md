# HireTrace API (Backend)

The HireTrace backend is a robust RESTful API engineered with **Node.js, Express, and MongoDB**. It serves as the data engine for the platform, securely managing users, authentication states, and interview experiences.

## 🏗 Architecture & Design

This backend follows the standard **MVC (Model-View-Controller)** pattern combined with independent route handlers and centralized error middleware.

### Folder Structure
- **`src/models/`**: Mongoose schemas defining the structure of exactly what gets saved to MongoDB (`user.model.js`, `experience.model.js`, `comment.model.js`). Features advanced Mongoose virtuals and indexes.
- **`src/controllers/`**: The core business logic. All database querying, data validation, and response formatting happens here.
- **`src/routes/`**: Express routers that map HTTP methods and endpoint paths to their respective controllers.
- **`src/middlewares/`**: Security and utility interceptors, including `protect` (JWT verification) and global `errorHandler`.

## 🔐 Authentication Ecosystem

HireTrace does not rely on third-party auth platforms (like Firebase). Instead, it implements a secure custom identity system:

1. **Password Security**: Passwords are never stored in plain text. They are hashed using `bcryptjs` with auto-generated salts.
2. **Access Tokens (JWT)**: Short-lived tokens (15m) generated by `jsonwebtoken` and passed to the frontend to authorize API requests.
3. **Refresh Tokens**: Long-lived tokens (7d) securely stored in HTTP-Only, `sameSite: strict` cookies, rendering them entirely immune to XSS attacks.
4. **Email Verification**: Uses `nodemailer` to dispatch uniquely generated cryptographic hex tokens to new users.

*Note on Deployment: The live Render deployment bypasses the strict email verification blocks due to Render Free Tier firewall restrictions blocking outbound SMTP packets.*

## 🛣 Core REST Endpoints

### Auth (`/api/auth`)
- `POST /register`: Creates a newly hashed user and sends verification email.
- `POST /login`: Validates credentials and attaches JWT refresh cookie.
- `POST /refresh`: Verifies refresh cookie and generates new access token.
- `GET /me`: Authenticated route to retrieve the current user's populated data.

### Experiences (`/api/experiences`)
- `GET /`: Retrieves a paginated, globally filtered list of interview experiences.
- `POST /`: Creates a new experience document.
- `PUT /:id/vote`: Toggles a user's upvote/downvote on a specific post.
- `PUT /:id/bookmark`: Adds or removes an experience from a user's saved list.

## 🚀 Environment Variables
To run this API locally, you must provide a `.env` file at the root of `server`:

```env
PORT=5001
MONGO_URI=mongodb+srv://...
ACCESS_TOKEN_SECRET=your_super_secret_jwt_key
REFRESH_TOKEN_SECRET=your_super_secret_refresh_key
CLIENT_URL=http://localhost:5173
EMAIL_USER=your_email@gmail.com
EMAIL_PASS=your_gmail_app_password
```

## 🛠 Running the Server
```bash
# Install dependencies
npm install

# Start development server with automatic nodemon restarts
npm run dev

# Start production server
npm start
```
